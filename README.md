# HospiPlan — Projet Bootcamp

Documentation et guides pour le projet **HospiPlan** (planification du personnel soignant, hôpital fictif Al Amal).

## Contenu du dossier

| Fichier | Description |
|--------|-------------|
| **HospiPlan_Guide_Etudiant (1).html** | Guide étudiant complet (3 phases : BD, API/contraintes dures, molles & génération). Ouvrir dans un navigateur (double-clic ou « Ouvrir avec »). |
| **HospiPlan_Guide_Etudiant_Allege.html** | Même présentation, **périmètre minimal** pour limiter la charge de travail. |
| **Image/** | Visuels utilisés par les guides (noms normalisés, voir ci-dessous). |

## Dossier `Image/`

Les fichiers du site ont été regroupés ici avec des noms stables (sans espaces, préfixe `hospiplan-`) :

| Fichier | Usage dans le guide |
|---------|---------------------|
| `hospiplan-preambule-hopital.jpg` | Préambule — contexte hôpital |
| `hospiplan-phase1-base-donnees.jpg` | Phase 1 — illustration BD / modèle de données |
| `hospiplan-phase2-architecture-fullstack.jpg` | Phase 2 — schéma frontend / API / base |
| `hospiplan-phase3-planning-optimisation.png` | Phase 3 — contexte planning / optimisation |
| `hospiplan-phase3-equipe-soins.jpg` | Phase 3 — bas de section / équipe soins |

Les chemins dans les **deux guides HTML** pointent vers `Image/nom-du-fichier` (même dossier que les fichiers `.html`).  
Si vous déplacez le HTML, conservez le dossier **Image** au même niveau que le fichier HTML (chemins relatifs).

## Autres fichiers à la racine

D’autres images ou ressources peuvent rester à la racine (non utilisées par le guide principal) : vous pouvez les archiver ou les supprimer selon vos besoins.

## Prérequis

- Navigateur récent (Chrome, Firefox, Edge).
- Aucun serveur obligatoire pour lire le guide (fichier statique).
